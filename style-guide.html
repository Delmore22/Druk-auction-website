Usage Notes
Use branded modals for validation messages, upload notices, success feedback, and discard confirmations instead of native browser alert() and confirm() dialogs.
Keep the eyebrow label set to Collectors Alliance Exchange so overlays feel like part of the product, not browser chrome.
Use the dark glass backdrop plus the light elevated dialog surface for all blocking overlays on owner-facing pages.
Primary modal actions should use the accent button treatment. Secondary actions should use the neutral cancel button style.

Usage Rules

Always Do
Always try Font Awesome first for any icon need. Only use another icon solution if Font Awesome 6 Free does not have a suitable icon for the use case.
Use var(--accent-color) for gold accents — never hardcode #DAA520 on its own anywhere that needs theming.
Use var(--border-color) for all borders. Never use raw hex border colors in new components.
Use var(--text-color) and var(--secondary-text) for all body text.
Read this file before writing CSS for a new page or component.
Add transition: all 0.3s ease (or specific properties) on interactive elements.
Follow existing border-radius sizes — pick the closest match from the Radius Scale section.
Sidebar icons always use the gold gradient background: linear-gradient(135deg, #DAA520, #FFD700).
Interactive rows (quick-action, record-item) hover with var(--accent-rgba) background.
Never Do
Do not override --accent-color at the page level unless the page has a completely different brand identity.
Do not use rgba(255,255,255,0.x) backgrounds for solid cards in light theme — use the surface tokens instead.
Do not add new status colors without adding them to this style guide.
Do not use inline style="" color values on production pages — those values belong in CSS with a token.
Do not use blue (#0078d4) as the page accent color — it has been superseded by site-global gold.
